# 刘看山 3D 建模验收清单

验收重点是还原 `source-image` 中的刘看山样图。模型必须读作白色北极狐简笔画角色：一体化白色身体、长鼻吻、黑椭圆鼻头、尖耳、点眼、小圆尾和黑色线状四肢。

## 形体

- [ ] 角色是白色北极狐 IP，不是狗、狼、写实狐狸或无鼻吻胖球。
- [ ] 头部和躯干是一体化大形，没有明显脖子和写实动物胸腔。
- [ ] 侧面鼻吻长而平，鼻头位于最前端。
- [ ] 鼻头为黑色圆角体，侧面明显大于正面。
- [ ] 眼睛为黑色点眼或短弧眼，没有复杂眼白和虹膜。
- [ ] 尖耳尺寸克制，从头顶轮廓自然长出。
- [ ] 尾巴为小圆尾或短弯尾，贴近身体后侧，不做成大蓬尾。
- [ ] 四肢保持黑色线状观感，并能对应骨骼和挂点。

## 3D 结构

- [ ] 模型原点在脚底中心。
- [ ] 默认正面朝向摄像机。
- [ ] 模型可 360 度旋转，正面、侧面和背面都有稳定识别度。
- [ ] 拓扑足够简单，适合实时渲染。
- [ ] 主体网格保持圆润干净，不出现写实肌肉、毛发和复杂褶皱。
- [ ] 鼻吻、鼻头、尾巴和四肢有足够结构支持动画。
- [ ] 手臂和腿部有最小厚度，避免在 Three.js 中因线条过细产生闪烁。
- [ ] 背面不显示鼻子，能清楚看到双耳、背部轮廓、尾巴和腿部落点。

## 骨骼

- [ ] 包含 `root`、`body`、`head`、`snout`、`tail`。
- [ ] 包含 `arm-left`、`arm-right`、`leg-left`、`leg-right`。
- [ ] 挂点包含 `head`、`mouth`、`hand-left`、`hand-right`、`tail`、`feet`、`emotion`。
- [ ] `head` 控制头部朝向时，不破坏头身一体的外轮廓。
- [ ] `snout` 能做轻微摆动或拖拽反馈，不导致鼻头脱离鼻吻。
- [ ] `tail` 能做低幅摆动、收起和上扬。
- [ ] 权重不会导致鼻吻、鼻头、尾巴或四肢变形异常。

## 材质

- [ ] 主体为白色或微暖白。
- [ ] 鼻头、眼睛、四肢和主要线条为黑色。
- [ ] toon 材质干净，避免写实光泽、皮革质感和毛发质感。
- [ ] 黑色描边清晰，粗细稳定。
- [ ] 外轮廓比内部线条更醒目，符合样图的粗线简笔画风格。
- [ ] 道具和情绪符号使用少量高饱和色，不抢主体识别。

## 动画

- [ ] `idle` 有轻微呼吸和小尾巴摆动。
- [ ] `walk` 和 `run` 的脚步落点稳定，细腿不穿模。
- [ ] `blink` 使用点眼到短弧眼或闭眼的低复杂度变化。
- [ ] `happy` 可挥手、比心或跳起，不破坏角色比例。
- [ ] `dragging` 有鼻吻或身体被拉动的方向感，四肢有滞后反馈。
- [ ] `hungry`、`sleepy`、`sick` 情绪清楚，主要依靠姿态和少量挂载特效表达。
