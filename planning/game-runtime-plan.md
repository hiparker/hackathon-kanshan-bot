# 刘看山游戏化客户端运行时方案

刘看山角色运行时改为 Three.js 承载。React 负责 UI 面板、状态展示和业务入口。Three.js 负责浏览器内 3D toon 角色渲染、方向旋转、动作状态、道具挂点和情绪特效。后端、AI、匹配、推荐和模型服务由其他项目处理。

## 技术分层

| 层级 | 技术 | 职责 |
| --- | --- | --- |
| 角色运行时 | Three.js | 渲染刘看山 3D toon 角色、播放动作、处理方向旋转、道具挂点、特效和角色事件。 |
| 角色资产 | 程序化几何、Blender、glTF/GLB | 先用程序化低模跑通交互，后续替换为 Blender 精模和骨骼动画。 |
| UI 层 | React、TypeScript | 状态面板、任务面板、道具栏、提醒卡片和 Demo 页面。 |
| 通信层 | kanshan-bridge | 定义 UI 与角色运行时的命令和事件协议。 |
| 客户端壳 | Web 容器或桌面壳 | 承载 React UI 与 Three.js canvas。 |

## 运行时边界

Three.js 负责所有角色本体能力。React 不直接操作几何体、材质、动画帧或 Three.js scene。React 只通过高层命令控制角色，例如设置方向、播放动作、装备道具和展示特效。

本仓库只处理客户端、UI、素材规范和通信协议。本仓库不实现后端接口、AI 模型、匹配算法、推荐算法、内容召回和真实数据服务。

## 资产流程

1. 在 `assets/concept` 中维护三视图、比例规范、动作清单和挂点说明。
2. 第一阶段用 `@kanshan/three-runtime` 的程序化低模跑通实时预览。
3. 第二阶段根据 `source-image` 样图调整比例、鼻吻、耳朵、尾巴、四肢和表情。
4. 第三阶段在 Blender 中制作修身北极狐精模。低模指适合实时渲染的低面数模型。
5. 为模型绑定骨骼。骨骼至少包含头、鼻吻、身体、尾巴、左右手、左右脚。
6. 制作第一批动画片段：`idle`、`walk`、`run`、`blink`、`happy`、`dragging`、`hungry`、`sleepy`、`sick`。
7. 导出 `.glb`，并在 Three.js 中通过 `AnimationMixer` 播放真实动画。

## 阶段路线

| 阶段 | 目标 | 验收标准 |
| --- | --- | --- |
| 阶段 1 | 建立 Three.js 运行时闭环。 | 浏览器首屏能看到 3D 刘看山，能响应动作、方向、道具和特效。 |
| 阶段 2 | 根据样图优化程序化模型。 | 模型保持刘看山北极狐识别度，正面和侧面比例正确。 |
| 阶段 3 | 接入 Blender 精模和 GLB。 | 程序化模型可被精模替换，桥接协议不变化。 |
| 阶段 4 | 接入真实骨骼动画。 | 第一批动作由 animation clip 驱动，事件回传稳定。 |
